# Forward Declarations

## One Definition Rule

### 1. Within a *file*
- Functions, variables, types, or templates in a given scope may only have one definition.

```cpp
int main() {
  int x{};
  // violate ODR 1
  int x{5};

  int y{add(6, 7) - subtract(6, 7)};
}
```

- ``g++ odr.cpp`` results in a compiler error.

### 2. Within a *program*
- Functions or variables in a given scope can only have one definition.

```cpp
//odr.cpp

// Violates ODR 2
int subtract(int x, int y) { return x - y; }

int main() {
  int y{add(6, 7) - subtract(6, 7)};
}
```

```cpp
// odr2.cpp

// Violates ODR 2
int subtract(int x, int y) { return x - y; }
```

- ``g++ odr.cpp odr2.cpp`` results in a linker error.

### 3. Duplicate but Equal Definitions
- Types, templates, inline functions, and inline variables can have duplicate definitions in different files, if the definitions are identical.

## Questions
> **Question 1:** What is a function prototype?
> *A function prototype is a declaration of a function that contains it's return type, identifier, and parameter types (optionally parameter names)*

> **Question 2:** What is a forward declaration?
> *A forward declaration is where you declare something, like a function prototype, before it is defined so that code that references it won't cause an error.*

> **Question 3:** How do we declare a forward declaration for functions?
> *A forward declaration for a function is a statement that contains the return type, function name, and parameters (it is the function prototype).*

> **Question 4:** Write the function declaration.
```cpp
int doMath(int first, int second, int third, int fourth) {
  return first + second * third / fourth;
}
```
> *Answer:*
```cpp
int doMath(int first, int second, int third, int fourth);
```
> **Question 5:** For each of the programs, state whether they fail to compile, fail to link, or ocmpile and link successfully.

> **Program A**
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y)
{
    return x + y;
}
```
> *It won't compile. There is no declaration/definition for add(int,int,int)*

> **Program B**
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
> *It won't compile. add(int,int,int) not declared before it's referenced.*

> **Program C**
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 = " << add(3, 4) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
> *It will fail to link, definition for add(int,int) won't be found.*

> **Program D**
```cpp
#include <iostream>
int add(int x, int y, int z);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int z, int y, int x) // names don't match the declaration
{
    return x + y + z;
}
```
> *It will compile. Names don't have to match, add(int,int,int) is declared and defined.*

> **Program E**
```cpp
#include <iostream>
int add(int, int, int); // no parameter names

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
> *It will compile. Parameter names in prototype are optional.*

> **Program F**
```cpp
#include <iostream>

int add(int x, int y);

int add(int x, int y, int z)
{
    return x + y + z;
}

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}
```
> *It will compile, functions with the same name but different parameters are considered to be different functions, no ODR violated.*
